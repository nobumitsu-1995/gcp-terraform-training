# Level 1 アーキテクチャ詳細

## 構成図

```
                   ┌─────────────┐
   ユーザー ──HTTPS──▶│  CloudFront  │  ← エッジロケーションでキャッシュ
                   └──────┬──────┘
                          │ Origin Access Control (OAC, SigV4署名)
                          ▼
                   ┌─────────────┐
                   │  S3 バケット  │  ← 非公開（CloudFrontからのみ読み取り可）
                   │ index/404.html│
                   └─────────────┘
```

## コンポーネント

1. **S3 バケット**: 静的ファイル（HTML/CSS/JS/画像）を格納。パブリックアクセスは全面ブロック。
2. **Origin Access Control (OAC)**: CloudFront が S3 を読むための署名付き認可。これによりバケットを公開せずに配信できる。
3. **CloudFront Distribution**: CDN。エッジでキャッシュし、HTTPSを終端し、エラーページ（403/404 → 404.html）を返す。
4. **S3 バケットポリシー**: 「このCloudFrontディストリビューション (SourceArn) からの `s3:GetObject` のみ許可」を表現。

## トラフィックフロー

```
1. ユーザーが CloudFront のドメインに HTTPS アクセス
2. CloudFront がエッジキャッシュを確認
   - ヒット: エッジから即返却
   - ミス: OAC署名付きで S3 オリジンから取得してキャッシュ
3. 存在しないパスは 403/404 → 404.html を返却
4. ユーザーにコンテンツ返却
```

## コスト注意点

- **CloudFront**: 1 TB/月のアウト転送が常時無料枠。研修規模なら $0。
- **S3**: ストレージ量（数MB）とリクエスト課金のみ。ほぼ $0。
- 静的サイトでは **ロードバランサー (ALB) を使わない**ため、GCP版のような時間課金のLBコストは発生しません。

⚠️ それでもワークショップ終了後は `terraform destroy` でリソースを片付ける習慣をつけましょう。
